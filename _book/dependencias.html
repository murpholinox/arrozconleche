2 Dependencias

En caso de falla de mi sistema operativo (SO) o si alguien desea reproducir mis resultados se listan las dependencias usadas.

2.1 Sistema operativo

El sistema operativo usado es Fedora 32 en su versión Workstation.

uname -a
## Linux eva02 5.6.12-300.fc32.x86_64 #1 SMP Mon May 11 16:47:13 UTC 2020 x86_64 x86_64 x86_64 GNU/Linux

2.2 Minería de datos

Para la extracción de datos se usa gemmi. Cuya instalación depende a su vez de git y cmake.

sudo dnf -y install git cmake
cd; mkdir Repos; cd Repos
git clone https://github.com/project-gemmi/gemmi.git
cd gemmi
cmake .
make
echo "# Gemmi" >> ~/.bashrc
echo "alias gemmi='/home/murphy/Repos/gemmi/gemmi'" >> ~/.bashrc

Y para la transformación y limpieza de datos se usan grep, awk y sed que vienen instalados por defecto en Fedora 32. También se utiliza: R, en especial varias funciones del conjunto de paquetes denominado tidyverse.

La instalación de R conviene hacerla como lo describo en mi blog.

La instalación del tidyverse depende a su vez de la instalación de las siguientes librerías en el SO.

sudo dnf -y install openssl-devel libxml2-devel libcurl-devel

2.3 Bitácora

La reproducción correcta, desde la ejecución del código hasta la producción del archivo de salida, de esta bitácora depende de la instalación de varios programas:

  • rstudio
  • bookdown
  • rmarkdown
  • renv

Siendo rstudio el entorno de desarrollo integrado de R y rmarkdown es un formato mezcla para documentar en markdown código en R (véase el sitio web de rmarkdown). El paquete bookdown ocupa la funcionalidad de rmarkdown para generar un libro en formato .Rmd que puede exportarse a varios formatos de salida (.html, .pdf, .doc, etc). Para manejar las dependencias de una manera aislada, portable y reproducible se usa renv.
